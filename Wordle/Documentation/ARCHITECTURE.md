# 🎮 Wordle - Архитектура проекта

## 📁 Структура папок

```
src/
├── main.js              # Точка входа приложения
├── core/                # Основная логика игры
│   ├── gameState.js     # Управление состоянием игры
│   ├── gameLogic.js     # Логика проверки угадания
│   └── stats.js         # Работа со статистикой (localStorage)
├── ui/                  # Компоненты пользовательского интерфейса
│   ├── board.js         # Отрисовка и управление доской
│   ├── keyboard.js      # Отрисовка и управление клавиатурой
│   ├── modal.js         # Модальные окна (статистика, новая игра)
│   └── theme.js         # Управление темой (светлая/тёмная)
├── input/               # Обработка пользовательского ввода
│   ├── keyboardHandler.js # Обработка физической клавиатуры
│   └── uiEvents.js      # Обработка событий UI (клики, ввод)
└── utils/               # Вспомогательные функции и константы
    ├── constants.js     # Константы, маппинги, конфигурация
    └── helpers.js       # Утилиты и вспомогательные функции
```

## 🏗️ Описание модулей

### `core/` - Основная логика

- **gameState.js** - Отвечает за состояние игры (текущая строка, угадание, загаданное слово)
- **gameLogic.js** - Проверка правильности угадания, определение побед
- **stats.js** - Сохранение и получение статистики из localStorage

### `ui/` - Отрисовка и стили

- **board.js** - Создание сетки 6x5, управление ячейками, отображение букв
- **keyboard.js** - Отрисовка виртуальной русской клавиатуры
- **modal.js** - Модальные окна (статистика, кнопка новой игры)
- **theme.js** - Переключение между светлой и тёмной темой

### `input/` - Обработка ввода

- **keyboardHandler.js** - Слушатель физической клавиатуры с маппингом English → Russian
- **uiEvents.js** - Обработчики кликов, введённых букв, отправки и удаления

### `utils/` - Утилиты

- **constants.js** - Слова, маппинг букв, раскладка клавиатуры, константы конфигурации
- **helpers.js** - Функции преобразования, показа сообщений, выбора случайного слова

## 🔄 Поток данных

```
index.html

src/main.js (инициализация)
    ↓
```

## 🎯 Основные функции

### Инициализация

- Загрузка состояния из localStorage
- Создание DOM элементов
- Привязка обработчиков событий

### Игровой процесс

1. Пользователь вводит букву (физ. клавиатура или виртуальная)
2. Буква отображается на доске
3. При заполнении всех 5 позиций автоматически проверяется угадание
4. Показывается цвет (зелёный/жёлтый/серый) для каждой буквы
5. Или начинается новая строка, или игра заканчивается

### Статистика

- Сохраняется количество выигрышей
- Сохраняется общее количество попыток
- Рассчитывается процент побед

## 🔄 Зависимости между модулями

- **main.js** зависит от всех модулей (импортирует всё)
- **input/uiEvents.js** зависит от core/, ui/, utils/
- **ui/modal.js** зависит от core/stats.js
- Остальные модули относительно независимы

## ✨ Преимущества такой архитектуры

- ✅ **Модульность** - каждый модуль отвечает за одно
- ✅ **Переиспользование кода** - функции легко переиспользуются
- ✅ **Тестируемость** - каждый модуль можно тестировать отдельно
- ✅ **Масштабируемость** - легко добавлять новые фичи
- ✅ **Читаемость** - код разбит на понятные части с JSDoc
