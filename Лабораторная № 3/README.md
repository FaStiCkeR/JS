# **Лабораторная работа № 3**
## Заметки
Новый класс создается при помощи синтаксиса:
```javascript
class JSClass {
    constructor(argumentOne) {
        this.argumentOne = argumentOne;
    }
    
    methods();
}
```
В случае желания унаследовать поля и методы одного класса для другого, используется следующий синтаксис:
```javascript
class Inerhetion extends JSClass {
    constructor(argumentOne, argumentTwo);
    super(argumentOne);
    
    newMethods();
}
```
`extends` позволяет нам унаследовать от родительского класса поля и методы, но в случае необходимости добавления 
каких-либо значений используется ключевое слово super()

`?.` — это синтаксис опциональной цепочки, данный синтаксис используется для того, чтобы избежать ошибки в случае 
объявления возможно отсутствующих параметров объекта, игнорируя ошибку и выводя undefined, в случае отсутствия параметра. 
### Какое значение имеет this в методах класса?
Ключевое слово this в методах класса позволяет взаимодействовать с полями класса, для их добавления, изменения, 
обработки в классе. this ссылается на конкретный экземпляр класса, созданный при вызове конструктора
### Как работает модификатор доступа # в JavaScript?
Модификатор доступа `#` позволяет сделать определенные поля приватными или методы для запрета их пользования вне методов 
класса и обеспечивают настоящую инкапсуляцию.
### В чем разница между классами и функциями-конструкторами?
Классы и функции-конструкторы - это два способа создания объектов с прототипным наследованием, только вот классы в 
JavaScript является синтаксическим сахаром, у них отличительный синтаксис, функции-конструкторы имеют особенность 
hoisting, поднятия и видимость, независимо от объявления, при классе обязательно нужно использовать конструкцию new, 
когда как для функций-конструкций можно и не использовать new.
